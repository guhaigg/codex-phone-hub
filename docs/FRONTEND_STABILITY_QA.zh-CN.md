# 前端稳定性 QA

这份清单用于防止 Codex 远程工作台再次出现输入、搜索、切页、刷新后卡死的问题。

## 自动化

运行真实浏览器稳定性测试：

```bash
npm run test:e2e --workspace packages/codex-web
```

当前覆盖：

- 桌面端打开已有会话、输入、搜索、切到工作台、返回会话、回车发送。
- composer 快捷按钮更新输入框时不替换 `#prompt-input` DOM 节点。
- 手机尺寸下新建会话、输入、回车发送、刷新后继续可操作。
- 测试捕获 page error、HTTP 4xx/5xx、非 SSE 的 failed request。

## 手动验收

部署到 `https://cockpit.codexgu.website/` 后按顺序检查：

1. 登录后点侧边栏已有会话，输入框能立即聚焦。
2. 输入中文和英文，光标不跳，页面不整体闪烁。
3. 点输入框下方“总结改动”和“运行测试”，文字追加到输入框，输入框不失效。
4. 侧边栏搜索连续输入 5 次，聊天输入内容不丢。
5. 点“工作台”“设置”“会话”来回切换，页面仍能点击。
6. 手机浏览器新建会话、输入、回车发送，刷新后页面能回到可操作状态。
7. 浏览器 DevTools Console 不应出现未处理异常或 API 500。

## 前端红线

- 输入、聚焦、快捷按钮、搜索不得触发整页 shell 重绘。
- 后台刷新不得抢输入框焦点。
- 切页失败必须释放 loading 状态。
- PWA / service worker 不得缓存登录态 API 响应。
